# CoalPlan AI v0.1 运行手册

v0.1 的目标是形成稳定、可追溯、可人工介入的施组生成闭环：

`模板选择 -> 投标 Markdown 输入 -> 切章与来源目录 -> 项目概况 -> 可编辑目录 -> 字数控制 -> 来源映射 -> 逐章生成 -> 小节树审阅 -> 多版本选择 -> 合并 final.md`

当前暂不处理图纸生成、CAD、多模态图片理解和云端回写。

## 1. 环境配置

项目根目录已有 `.env`，默认使用 DeepSeek flash：

```env
COALPLAN_LLM_PROVIDER=deepseek
COALPLAN_STRUCTURED_LLM_PROVIDER=deepseek
COALPLAN_DEEPSEEK_BASE_URL=https://api.deepseek.com
COALPLAN_DEEPSEEK_MODEL=deepseek-v4-flash
COALPLAN_LLM_TRACE_DIR=.coalplan-llm-traces
```

不要把新的密钥写入文档；本地 `.env` 作为运行配置来源。

## 4. 推荐演示流程

1. 在左侧选择模板并创建项目。
2. 上传标准 Markdown 投标技术文件。
3. 点击生成基础目录，确认来源目录、项目概况和项目目录已出现。
4. 执行生成前目录精修，先查看 proposal，再确认应用。
5. 执行目标字数估算，必要时手动调整重点章节字数。
6. 在章节工作区补充人工要求、表格说明或附件说明。
7. 选择一个叶子章节执行单章生成。
8. 查看来源摘要、生成正文、人工补充、正文小节树、证据审计和组织模式审计。
9. 对需要修订的小节或章节执行建议动作，或手动编辑保存新版本。
10. 选择每章最终版本。
11. 合并生成 `final.md`。

## 5. 前端流水线总览应看什么

顶部流水线总览用于一眼判断项目处于哪个阶段：

- 项目：模板和项目是否已确定。
- 输入：是否上传投标 Markdown，切章数量是多少。
- 目录：项目可编辑目录节点数、是否存在待确认 proposal。
- 字数：多少节点已有目标字数，总目标字数是多少。
- 生成准备：readiness 总体状态、可自动执行动作和人工确认动作数量。
- 章节版本：已选中版本数量、当前章节版本数量。
- 合并：是否已有 final.md。

这个总览只负责展示和引导，不替代中部目录树和右侧章节工作区。

## 7. v0.1 验收标准

- 上传 Markdown 后能持久化输入、切章、来源目录。
- 生成目录后，用户能增删改查目录节点并调整四模块和目标字数。
- 目录精修、AI 修改目录、AI 修改章节都先形成 proposal，用户确认后才应用。
- 单章生成前能映射来源章节，并把来源全文、证据、补充材料、字数要求加入 prompt。
- 章节补充材料、附件说明、版本和选中版本刷新后仍存在。
- 单章生成后能查看正文小节树、来源链接、证据审计和写作模式审计。
- 门控结果作为建议，不强制替用户丢弃版本。
- 合并时只使用用户选中的版本。

## 8. 当前风险

- DOCX/PDF 转 Markdown 仍属于前置工具，核心 pipeline 只吃 Markdown。
- 图片附件第一版只使用用户说明，不做视觉理解。
- 高密度工艺章需要先拆小节再生成，整章长文本容易遗漏来源事实。
- 本地施组模式库只作结构和写法参考，不是事实来源。
- 真实 LLM 输出质量需要继续用 project3/project4 小样本回归，不应只依赖 fake LLM。
